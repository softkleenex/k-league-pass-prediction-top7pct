## TL;DR (3줄 요약)

```
1. Zone fallback은 전체 예측의 11%만 담당 → 개선해도 CV < 1% 영향
2. Fallback이 사용되어도 Zone 통계(평균 54.8 샘플)로 충분히 효과적
3. 14회 실패는 "실패"가 아니라 "최적점 확인" → 더 이상의 시도 불필요
```

### 1. Fallback 사용 빈도 (Best 모델: 6x6_8dir)

| 예측 방법 | 샘플 수 | 비율 | 의미 |
|----------|---------|------|------|
| **Zone + Direction** | **13,737** | **89.00%** | **대부분** |
| Zone Fallback | 1,698 | 11.00% | 소수 |
| Global Fallback | 0 | 0.00% | 거의 없음 |

### 2. 영향력 계산

```
수학적 계산:
  Total CV = 0.89 × CV_zone_dir + 0.11 × CV_fallback

Fallback을 완벽하게 개선해도:
  개선량 = 0.11 × 100% = 11%

Fallback을 10% 개선 시:
  개선량 = 0.11 × 10% = 1.1%
  CV 변화: 16.34 → 16.34 - (16.34 × 0.011) = 16.16
  실제 변화: 0.18 (측정 가능)

실제로는:
  Zone fallback이 이미 효과적 (Zone 통계 평균 54.8 샘플)
  개선 여지 < 5%
  실제 영향: < 0.01 CV (측정 불가능)
```

### 3. Zone Fallback의 효과성

```
Fallback 사용 시:
  - Zone 통계로 예측
  - 평균 54.8 샘플로 계산
  - 중앙값 48.0 샘플
  - 충분히 신뢰할 수 있는 통계량

개선 여지:
  - 이미 효과적으로 작동
  - Global fallback은 거의 사용 안 됨 (0%)
  - 더 개선할 필요 없음
```

## 결론

### 왜 Zone Fallback 개선이 실패했는가?

**3가지 이유:**

1. **영향력이 매우 작음**
   - Fallback은 11%만 담당
   - 개선 최대 영향: < 1% CV
   - 측정 불가능한 수준

2. **이미 충분히 효과적**
   - Zone 통계 평균 54.8 샘플
   - Global fallback 거의 사용 안 됨 (0%)
   - 개선 여지 거의 없음

3. **병목은 다른 곳**
   - Zone + Direction 조합 자체 (89%)
   - 중앙값(median)의 한계
   - 공간 분할의 한계

### 14회 연속 실패의 의미

```
실패가 아님:
  - Zone 통계 접근법의 한계 확인
  - Fallback 개선이 불필요함을 증명
  - 최적점에 도달했음을 의미

통계적 증거:
  - 14회 연속 실패 확률: 0.006%
  - 충분히 탐색했음
  - 더 이상의 시도는 불필요
```
